# Übung 2: Mini-Spec schreiben (Tag 1)

## Ziel
Aus einer vagen Idee eine präzise Spezifikation (Spec) im Team erarbeiten, die morgen an einen Coding-Agenten übergeben werden kann.

## Aufgabe
Wählt eines der folgenden Mini-Features oder eine eigene Idee:
- **Option A**: Registrierungsformular mit Feld-Validierung
- **Option B**: Einfache REST-API zur Datenabfrage (z.B. Produktkatalog)
- **Option C**: Taschenrechner mit Verlauf (Speicherung im RAM)
- **Option +**: Eine eigene kreative Idee!

## Ablauf (35 Minuten)
1. Füllt die sechs Felder in der Vorlage `TEMPLATE_SPEC.md` vollständig aus.
2. **Rollen**: Ein *Schreiber* (formuliert die Details) und ein *Skeptiker* (hinterfragt Annahmen, sucht nach Lücken). Alle 10 Minuten tauschen!
3. **DoD (Definition of Done)**:
   - Alle 6 Felder sind befüllt.
   - Nicht-Ziele sind explizit benannt (was soll die App *nicht* tun?).
   - Ein Dritter (oder die KI) kann das Feature ohne Rückfragen vollständig implementieren.

## Dateien
- `TEMPLATE_SPEC.md` (Kopiert diese Datei und befüllt sie für euer Feature!)
