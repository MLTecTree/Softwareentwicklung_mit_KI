# Übung 5: Setzt eure Mini-Spec um (Tag 2)

## Ziel
Den kompletten Software-Entwicklungszyklus mit einem Coding-Agenten von Anfang bis Ende durchspielen: Spec -> Glossar -> KI-Entwicklung -> Tests -> Commit.

## Aufgabe
Nehmt die Mini-Spec, die ihr in Übung 2 geschrieben habt, und setzt sie jetzt vollständig in Java um!

## Ablauf (40 Minuten)
1. **Verzeichnis & Git**: Erstellt eine Standard-Maven-Struktur in diesem Ordner (`pom.xml`, `src/main/java`, `src/test/java`).
2. **Fachbegriffe sichern**: Erstellt eine `GLOSSARY.md` mit den Kernbegriffen aus eurer Spec.
3. **Agenten steuern**: Übergebt eure Spec als Prompt an OpenCode.
4. **Interaktive Unterstützung**: Nutzt bei Bedarf die Skills `/grill-me` (für Architektur-Diskussionen) oder `/tdd` (für die testgetriebene Umsetzung).
5. **Qualität sichern**: Lasst Maven-Tests laufen (`mvn test`) und korrigiert Fehler. Committet funktionierende Zwischenstände.
6. **DoD (Definition of Done)**:
   - Eure Spec ist vollständig implementiert.
   - `GLOSSARY.md` existiert im Projektordner.
   - Alle Tests sind grün.
   - Ein fertiges, funktionierendes Java-Feature ist im Git-Repository committet.
